Process Step

- Login to admin dashboard.
- Search the MID in the user entity and open the merchant ID details page.
- Find "Terminals".
- Check if Paypal terminal is available.

IF Terminal = Available:

Process Step

- Open the terminal ID and check the following parameters:
- Enable Wallets paramter contains : ["paypal"]
- Enabled : Tick/True
- Status: Activated

IF all parameters = Valid:

Ticket Action:

- End State Action: Resolution By The Same Group
- Agent Category: TS-Dashboard
- Agent Sub-Category: Dashboard configuration issue
- Agent Item: Paypal Integration
- Click on UPDATE button.

Instruction

Inform the merchant that they need to login to their paypal account/dashboard and check for any notifications from paypal
propmpting them to enable razorpay.

Ticket Action: Update Status: to "Resolved"

Process Step

- login to admin dashboard with the MID.
- Open inspect- network call, try to login to the MID from the admin dashboard.
- Find the activation api call response.
- Within the API response find "required field" parameter.

- End State Action: Resolution By The Same Group
- Agent Category: TS-Dashboard
- Agent Sub-Category: Activation/Onboarding
- Agent Item: Unable to Submit KYC
- Click on UPDATE button.

Instrcution

Inform the merchant that the following details { parameters/fields in required field parameter} are not uploaded correctly and need to be re-uploaded.

Ticket Action: Update Status: to "Resolved"



Process Step

- Login to admin dashboard.
- Go to merchant detail from entities and search the merchant ID in Search Entity Id.
- Find the parameters: Promoter Pan & Pan Name.

IF PAN name = GST/CIN signatories

- End State Action: Resolution By The Same Group
- Agent Category: TS-Dashboard
- Agent Sub-Category: Activation/Onboarding
- Agent Item: GST / PAN Validation Issues
- Click on UPDATE button.

Process Step

Inform the merchant of the activation status of their account and ask them to reachout if there are any further queries.

Ticket Action: Update Status: to "Resolved"



IF PAN name != GST/CIN signatories

- End State Action: Resolution By The Same Group
- Agent Category: TS-Dashboard
- Agent Sub-Category: Activation/Onboarding
- Agent Item: GST / PAN Validation Issues
- Click on UPDATE button.

Instruction

Mx to needs to update the GST/CIN certificate and have the promoter pan name as one of the signatories.

Note: If the merchant is not willing to follow the above step, as an alternative, mx can create a new MID using pan
details of one of the signatories present in GST/CIN certificate.

Ticket Action: Update Status: to "Resolved"

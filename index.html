IF merchant query = unclear/unavailable AND Payment ID/UTR = Available:

Process Steps:

- Login to the SAV dashboard.
- Search with the payment ID in the search bar.
- Check the status of the payment.

IF Payment status = Failed

Refer to "Transaction Failure --> IF Payment ID/UTR = Available:" flow.

IF Payment status = Anything apart from "Failed"

Ticket Action:
End State Action: Additional Info Required
Click on UPDATE button.


Instruction:

Inform the merchant that the payment ID is currently in {status of the payment} status. Inform the merchant they can reachout incase there is any additional query.

  • - When the merchant raises a query that payments are moving from failed to authorised.
    - When the payment is initally marked as failed but after sometime it is moving to authorised state.
  • - When the merchant raises a qury that the customer is debited but the merchant did not receive the payment.
    - When the customer has made the payment but the payment ID is not visible in the merchant dashboard.
  • - When then merchant raises a query that their transaction is failing.
  • - When the merchant raises a query that their payments are not getting auto captured.
    - When the payments are getting auto refunded.
  • - When the merchant informs that they are unable to create QR on their account.
  • - When the merchant informs that QR is visible on dashboard but not visible on mobile.

IF Payment ID/UTR = Available:

Process Steps:

- Login to the SAV dashboard.
- Search with the payment ID in the search bar.
- Once the payment details populate on the screen, click on ""show more"" button to populate all the fields.
- Check the status of the payment.
- Search for the Late auth parameter

IF Late Auth = True:

Ticket Action:

- End State Action: Resolution By The Same Group
- Agent Category: TS-PG-UPI
- Agent Sub-Category: Capture Failure
- Agent Item: Late Auth
- Click on UPDATE button.

Communication:

We would like to inform you that we have received a delayed response from the bank due to an issue on the customer's bank side.
This scenario is called Late Authorisation of payments. To handle such cases effectively, we request you to implement the following:
Fetch API and Webhooks. For more information, refer to Late Authorised Payments.

Ticket Action: Update Status: to "Resolved"

IF Late Auth = FALSE:

Steps:

Compare the created time of the payment vs the authorised time of the payment.

IF (auth time - created time) < 12 mins:

Ticket Action:

- End State Action: Resolution By The Same Group
- Agent Category: TS-PG-UPI
- Agent Sub-Category: Capture Failure
- Agent Item: Late Auth
- Click on UPDATE button.

Instruction:

Inform the merchant that the razorpay received delayed success response from the gateway due to which this scenario happened.
Also inform the merchant that delay is well within the expected payment completion time. In order to handle such scenarios the merchant
needs to implement Fetch API and Webhooks. For more information, refer to Late Authorised Payments.

Ticket Action: Update Status: to "Resolved"
